1 Why to compress?

  • Data compression can reduce the memory requirements of (almost) any kind of information source.

2 Which data?

  • Mainly … audio, image, and video signals.

3 Why these sources?

  • After the digitalization of any signal we get a sequence s[] of samples that represent the signal s with more or less ﬁdelity.
  • Usually, s[] is encoded using PCM (Pulse Code Modulation), in which every sample s[i] is represented with the same number of bits.
  • Most digital PCM signals are memory demanding. For example, in a CD we have a data-rate of
(16 + 16) bits sample × 44.100samples second = 1.411.200 bits second. (1)
  • Image and video signals require much more memory.

6 Memory requirements of PCM video

  • In RGB (PCM) video, each color pixel need at least 24 bpp (bits/pixel).
  • The memory requirements of RGB video are enormous. For example, an hour of 640 × 480 × 25 Hz true-color of PCM video needs:
25images second × 640 ⋅ 480pixels image × 24 bits pixel = 184.320.000 bits second (2)
184.320.000 bits second × 3.600seconds hour × 1G 1.0243 ×1byte 8bits ≈ 77Gbytes (3)
  • Video coding techniques should be used to compress this data. Most of these techniques are bases on Block-based Motion Estimation.
